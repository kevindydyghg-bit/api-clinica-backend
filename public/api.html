Guía para pruebas (Postman / Thunder Client)

URL Base: https://api-clinica-backend.onrender.com

⚠️ Aclaraciones Importantes de Uso

  • La API es estricta: Si se intenta enviar una petición POST sin los campos obligatorios (nombre_completo, fecha_nacimiento, telefono), la API rechazará la petición con un error 400.
  • POST crea registros nuevos (No edita): Para editar la información de un paciente ya existente, la API cuenta con el método PUT.
  • ¿Cómo ver los cambios reflejados? Después de enviar datos exitosamente (201 Created), se puede verificar enviando una petición GET en Postman hacia la misma ruta, o recargando la tabla en el panel de administrador web.

GET /api/pacientes

Descripción: Obtiene la lista completa de todos los pacientes registrados.

Reglas: No requiere enviar ningún cuerpo (body) ni parámetros.

POST /api/pacientes

Descripción: Crea un nuevo registro de paciente en la base de datos.

Reglas (Headers): Debe incluir Content-Type: application/json.

Reglas (Body): Se requiere enviar un JSON estructurado de la siguiente manera:

{
  "nombre_completo": "Juan Pérez",
  "fecha_nacimiento": "1990-05-15",
  "telefono": "3141234567",
  "email": "juan@correo.com",
  "direccion": "Centro, Manzanillo",
  "notas_historial": "Paciente sin alergias conocidas."
}

PUT /api/pacientes/:id

Descripción: Actualiza la información de un paciente que ya existe.

Reglas (URL): Reemplazar :id por el ID del paciente a editar (ej. /api/pacientes/1).

Reglas (Body): Requiere el JSON completo con los datos actualizados.

{
  "nombre_completo": "Juan Pérez Modificado",
  "fecha_nacimiento": "1990-05-15",
  "telefono": "3149998888",
  "email": "juan_nuevo@correo.com",
  "direccion": "Nueva Dirección, Manzanillo",
  "notas_historial": "Actualizó su teléfono."
}

GET /api/productos

Descripción: Obtiene la lista del inventario médico.
